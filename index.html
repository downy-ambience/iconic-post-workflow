Workflow Architecture & Resolution Strategy

⚠ 현재 문제점 요약

Dakota의 편집본에 줌/포지셔닝/카메라쉐이크/스피드램핑이 적용된 상태에서, 이 효과가 VFX 트래킹 & 로토 작업을 방해함. 컬러리스트는 EDL 기반 원본 작업을 원하고, VFX팀(Giovi/DQ)은 클린 플레이트를 원하는 상충 발생.
STEP 01 — ON-SET
현장 편집 (On-Set Edit)
현장에서 촬영본을 바로 조합하여 1차 편집본 완성. 연출감독과 현장에서 실시간 논의하며 컷 확정.
👤 장진혁 감독 (Editor)
Premiere Pro / DaVinci — 원본 4608×3164 footage
STEP 02 — OFFLINE EDIT
오프라인 추가 편집 (LA)
Dakota가 코디와 함께 프레임 단위 수정 및 디지털 줌, 포지셔닝, 카메라 쉐이킹, 스피드램핑 효과 추가. 3840×2160 시퀀스 기준으로 작업하여 원본 4608×3164 대비 충분한 해상도 여유 확보.
👤 Dakota (LA Editor) + 코디
저해상 MP4 프록시로 편집 → 3840×2160 sequence
STEP 03 — RELINK
원본 Relink & EDL 출력
Dakota가 4444XQ 다운로드 불가 → 저해상 MP4로 편집한 프로젝트를 진혁이가 받아서 원본 푸티지(4608×3164)로 Relink. 줌/포지셔닝/스피드램핑 효과를 제거하고 클린 EDL로 컬러실에 전달.
👤 장진혁 감독
EDL export — effects stripped
STEP 04 — COLOR
컬러 그레이딩
컬러리스트가 EDL로 원본 4608×3164 푸티지를 불러와서 풀 해상도 컬러 작업 진행. 컬러된 푸티지를 팀에 전달.
🎨 Colorist
DaVinci Resolve — 4608×3164 원본 기준 컬러
STEP 05 — ⚠ BOTTLENECK
컬러본 재편집 → VFX 전달
컬러된 푸티지를 받아서 진혁이가 다시 편집본 재구성. Dakota의 줌/쉐이크/스피드램핑을 다시 적용하여 4444XQ MOV 출력 → VFX팀 전달.

문제: VFX팀이 트래킹/로토 작업 시 줌·쉐이크·스피드램핑이 적용된 상태면 작업 난이도와 시간이 급격히 증가. Giovi가 "효과를 파이널 컨폼 단계로 미뤄달라"고 요청한 이유.
👤 장진혁 → 🔧 DQ (VFX Lead) / Giovi (Post Producer)
출력: 4444XQ MOV — 줌/쉐이크 포함 여부가 핵심 이슈
STEP 06 — VFX
VFX / 3D 작업
DQ와 VFX팀이 컬러된 편집본 기반으로 합성, 3D, 로토스코핑, 트래m�� 등 진행. 해상도가 클수록 렌더 시간↑, 작업 복잡도↑.
🔧 DQ (VFX Lead) + VFX Team
Nuke / After Effects / 3D Software
STEP 07 — CONFORM
파이널 컨폼 & 마스터링
VFX 완성본 + 컬러본을 합쳐서 최종 마스터. 줌/쉐이크/스피드램쥑 효과를 이 단계에서 적용할 것인지가 핵심 의사결정 포인트.
👤 장진혁 / Conform Editor
최종 출력: 3840×2160 마스터
STEP 08 — DELIVERY
최종 딜리버리
마스터링 완료 후 각 플랫폼별 스펙에 맞춰 최종 파일 출력 및 납품.
📦 Ambience Inc.

✅ 권장안: "Clean Plate → VFX → Final Conform" 방식

Giovi의 제안을 수용하여, 줌/쉐이크/스피드램핑을 VFX 이후 파이널 컨폼 단계에서 적용하는 워크플로우. VFX팀에는 클린 플레이트만 전달하여 트래m��/로토 효율을 극대화.

💡 핵심 아이디어

진혁이가 컬러본을 받으면 Dakota의 효과(줌/쉐이크/스피드램핑)를 적용하지 않은 "클린 버전"으로 4444XQ MOV를 VFX팀에 전달. Dakota의 편집 의도(효과 데이터)는 별도로 보존하여 파이널 컨폼에서 적용.
  • VFX팀: 안정적인 클린 플레이트로 작업 → 트래킹/로토 정확도↑, 렌더시간↓
  • 파이널 컨폼: VFX 완성본 위에 줌/쉐이크/스피드램핑 최종 적용
  • 해상도: VFX는 3840×2160 기준 작업 가능 (EP 원안 반영)

현재 방식 vs. 권장 방식 (Giovi 제안 수용)

❌ 현재 방식 (효과 적용 후 VFX)

  1. 현장편집 → Dakota 효과 추가
  2. Relink → 클린 EDL 출력
  3. 컬러 그레이딩 (4608×3164)
  4. 컬런본에 효과 재적용 → 4444XQ 출력
  5. VFX가 효과 적용된 상태에서 작업
  6. 파이널 컨폼 → 둜리버리

✅ 권장 방식 (클린 플레이트 → VFX → 컨폼)

  1. 현장편집 → Dakota 효과 추가
  2. Relink → 클린 EDL + 효과 데이터 별도 보존
  3. 컬러 그레이딩 (4608×3164)
  4. 클린 플레이트로 4444XQ 출력 (효과 없이)
  5. VFX 클린 플레이트 기반 작업
  6. 파이널 컨폼에서 효과 최종 적용
  7. 딜리버리
해상도별 영향 분석
VFX 렌더시간 및 작업효율성에 미치는 영향
항목 4608×3164 (원본) 3840×2160 (UHD) 비고
픽셀 수 ~14.6M ~8.3M UHD가 43% 적음
렌더 시간 (예상) 기준 (1x) ~0.57x 항셀 비례 단숈 계산
디스크 용량 훈 약 43% 절약 4444XQ 기준
최종 둜리버리 품지 4K+ 여유 4K 정확 fit 최종이 3840 이면 차이 없음
줌/리프레임 여유 있음 (~120%) 없음 (1:1) 컨폼에서 줌 적용 시 원본이 유리

📐 해상도 결정 가이드

  • Dakota의 줌 정도가 큰 샷이 많다면: VFX도 4608×3164로 작업하는 것이 안전. 컨폼에서 줌 적용 시 해상도 손실 없음.
  • 줌이 미미하거나 대부분 쉘이크/스피드램핑 위주라면: 3840×2160으로 VFX 작업해도 무방. 렌더시간 대폭 절약.
  • 절충안: 줌이 큰 VFX 샷만 4608 기준, 나머지는 3840 기준 — 샷별 분류 필요.
